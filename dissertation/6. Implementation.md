## 6.1 Architecture Overview
  - METAFOR framework which includes LARA Framework under the hood
  - Consist of 3 elements
    - dumping source code IR in json format using llvm compiler flang22 and flang-dumper plugin implemented by the team
    - source code parsing and building Asymetric Syntax Tree (AST) - Java module
    - Weaver - java module that enables manipulation over AST using Node.js scripts
    - Fortran-JS - nodejs module used to write scripts for AST manipulation and transformation
## 6.2 Parsing, AST Representation, Weaving and Code Generation
  - to parse the initial code in this project we used llvm compiler flang22
  - to extract convinient data the plugin flang-dumper have been build that dump nodes in JSON format
    - the plugin needed to be adjusted to correctly fetch information from fotran nodes that has complex structure
  - from that point onward each node had their own rules and node represntation in fortran-transpiler AST
  - in order to manipulate over AST all possible AST nodes and their methods needed to be exposed to Fotran-JS Typescript Interface
    - Properties needed to be exposed are described in XML files
    - FortranWeaver generates automaticaly Typescript Interfaces and abstract classes after XML structure is complete
    - In order to link typescript interface for join points and AST nodes for each abstract class concrete class needs to manually implemented.
  - Loop Transformations Algorythm overview
    The algorithm first traverses the AST to identify candidate loop nodes. Once a candidate is found, the transformation verifies that the loop satisfies the required preconditions. If the checks succeed, the corresponding AST subtree is rewritten and passed to the code generation stage. If the checks fail, the original loop is preserved unchanged.
  - After all steps are completed modified AST is trversed and the fortran source code is generated based on attributes of each AST node and the place it is located
    
## 6.4 Loop Interchange

### 6.4.1 Preconditions
  - two perfectly nested loops, without additional operations betwen outer and nested loop
  - no triangular loops - inner loops must not depend on the outer loop variable
    - ##### Triangular inner bound references outer variable
      > The inner loop's lower or upper bound is an expression involving the outer loop variable. After interchange, that expression is literally copied into the new outer loop's bound — but the variable it references is now the inner variable, which is out of scope at the outer bound position. The runtime reads garbage from the stack, producing wrong output or a segfault.

      ```fortran
      ! Original:
      do j = 1, maxgrid
        do i = j, maxgrid   ! inner lower bound = j (outer variable)
          do cnt = 1, length
            diff(cnt, i, j) = sumTang(i, j)
          end do
        end do
      end do

      ! After interchange (wrong):
      do i = j, maxgrid     ! j is undefined here — reads garbage from stack
        do j = 1, maxgrid
          do cnt = 1, length
            diff(cnt, i, j) = sumTang(i, j)
          end do
        end do
      end do
      ```    
  - nested loop does not depend on the outer loop variable
      - ##### Outer variable appears in a nested loop bound inside the body
        > The outer and inner loop bounds are rectangular (neither references the other variable). However, the body contains a third nested loop whose bounds use the outer variable. After interchange, this nested loop still compiles correctly because it now uses what was the outer variable (now inner), but the change in evaluation order causes previously-unmodified array values to be read as already-modified, producing wrong results.

        ```fortran
        ! Original — (i outer, j inner):
        do i = 2, ni           ! outer — rectangular bound, no ref to j
          do j = 1, ni         ! inner — rectangular bound, no ref to i
            do k = 1, i - 1    ! k-loop bound uses outer variable i
              b(j, i) = b(j, i) + alpha * a(k, i) * b(k, j)
            end do             ! writes b(j,i), reads b(k,j)
          end do
        end do

        ! After interchange — (j outer, i inner):
        do j = 1, ni           ! outer (was inner)
          do i = 2, ni         ! inner (was outer)
            do k = 1, i - 1    ! still syntactically valid: i is now inner
              b(j, i) = b(j, i) + alpha * a(k, i) * b(k, j)
            end do
          end do
        end do
        ```
### 6.4.2 Transformation algorithm

    1. Detect a perfectly nested loop.
    2. Identify the outer and inner loop levels to be swapped.
    3. Check that the interchange does not violate dependencies.
    4. Swap the loop headers.
    5. Preserve the original loop body inside the new nesting order.
    6. Regenerate the transformed code.

### 6.4.3 Example
```fortran
! Before
do i = lo_i, hi_i
  do j = lo_j, hi_j
    body
  end do
end do
! After
do j = lo_j, hi_j
  do i = lo_i, hi_i
    body
  end do
end do
```

## 6.5 Loop Fusion
### 6.5.1 Preconditions
  - two adjacent loops with identical boundaries
  - write subscript contains no instance of the fusion variable → the same array element is modified on every fusion iteration. If the other loop reads that element, it will see an incomplete value.
    - ##### Reduction not yet complete
      > One loop accumulates a value into an array element over its entire iteration range. The next loop reads that element. After fusion, the reader sees a partial (incomplete) accumulation.
      
      ```fortran
      ! Fusion variable: j
      do j = 1, n
        tmp(i) = tmp(i) + a(j,i) * x(j)   ! A: tmp(i) grows with every j
      end do
      do j = 1, n
        y(j) = y(j) + a(j,i) * tmp(i)     ! B: needs final tmp(i)
      end do

      ! After fusion (wrong):
      do j = 1, n
        tmp(i) = tmp(i) + a(j,i) * x(j)   ! partial after j=1
        y(j)   = y(j)   + a(j,i) * tmp(i) ! reads partial tmp(i) — wrong
      end do
      ```  

  - array X written as X(inner_var, fv) in one loop and read as X(fv, inner_var) in the other — transposed cross-iteration dependency.
    - ##### Transposed subscript across outer iterations
      > Loop A writes an array at index `(inner, outer)` — filling one column per outer iteration. Loop B reads the same array at index `(outer, inner)` — reading one row per outer iteration. A row of the array spans many columns, which are only written in future outer iterations that haven't run yet.
      
      ```fortran
      ! Fusion variable: i
      do i = 1, n
        do j = 1, n
          a(j, i) = a(j, i) + u1(i)*v1(j)  ! A: writes column i of a
        end do
      end do
      do i = 1, n
        do j = 1, n
          x(i) = x(i) + beta * a(i, j)     ! B: reads row i of a
        end do                              !    a(i,2), a(i,3)... not written yet
      end do

      ! After fusion at i=3 (wrong):
      do i = 1, n
        do j = 1, n; a(j,i) += ...         ! writes a(*,3) — column 3
        do j = 1, n; x(i) += ... a(i,j)   ! reads a(3,*) — row 3
      end do                                ! a(3,1) and a(3,2) still have old values
      ```    
  - array X written with a subscript that contains the fusion variable but no inner loop variable (one element per fusion iteration), and read by the other loop with a subscript that contains an inner loop variable (reads across the full range within one fusion iteration).
    - ##### Write-back before all reads are done
      > Loop A reads an array across its full range in an inner loop. Loop B writes a new value back into the same array at the end of each outer iteration. After fusion, the write in iteration `p=1` corrupts the array element that loop A still needs to read in iteration `p=2`.
      
      ```fortran
      ! Fusion variable: p
      do p = 1, np
        do s = 1, np
          sumA(p) = sumA(p) + a(s) * cFour(p,s)  ! A: reads a(s) for ALL s
        end do
      end do
      do p = 1, np
        a(p) = sumA(p)                            ! B: writes a(p) — one element per iter
      end do

      ! After fusion (wrong):
      do p = 1, np
        do s = 1, np
          sumA(p) = sumA(p) + a(s) * cFour(p,s)  ! reads a(s=1) at p=2 ...
        end do
        a(p) = sumA(p)                            ! ... but a(1) was overwritten at p=1
      end do
      ```
### 6.5.2 Transformation algorithm
  1. Detect two adjacent loops.
  2. Check that both loops have the same iterator, bounds, and step.
  3. Check that merging the loop bodies does not violate dependencies.
  4. Create one loop using the shared loop header.
  5. Append the body of the second loop after the body of the first loop.
  6. Replace the two original loops with the fused loop.

### 6.5.3 Example

```fortran
  !Before:
  do i = 1, n
    a(i) = b(i)
  end do
  do i = 1, n
    c(i) = d(i)
  end do
 
  !After:
  do i = 1, n
    a(i) = b(i)
    c(i) = d(i)
  end do
```

## 6.6 Loop Fission

### 6.6.1 Preconditions
  - loop containing multiple statements
  - scalar written in an earlier stmt appears in a later stmt's code. After fission, the later loop sees only the scalar value from its own iteration — it cannot observe the value threaded from the earlier loop.
    - ##### Scalar temporary threading
      > A scalar variable is written in one statement and consumed in a later one. After fission, all iterations of the "producer" loop complete before any iteration of the "consumer" loop begins — the scalar holds only the producer's last-iteration value, so every consumer iteration reads the wrong value.

      ```fortran
      ! Original — nrm is computed fresh each k-iteration:
      do k = 1, nj
        nrm = 0.0D0                           ! S1: init
        do i = 1, ni
          nrm = nrm + a(k, i) * a(k, i)       ! S2: accumulate
        end do
        r(k, k) = sqrt(nrm)                   ! S3: consume ← depends on S1+S2
      end do

      ! After fission (WRONG):
      do k = 1, nj; nrm = 0.0D0; end do           ! loop 1
      do k = 1, nj; do i; nrm = nrm + ...; end; end ! loop 2 — nrm ends at k=nj value
      do k = 1, nj; r(k,k) = sqrt(nrm); end do   ! loop 3 — every k reads same stale nrm
      ```    

  - a later stmt writes an array that an earlier stmt reads. After fission, the earlier loop (for ALL iterations) runs before the later loop, so the earlier loop reads stale values for every iteration after the first one that the later loop would have updated.
    - ##### Later stmt writes array that earlier stmt reads
      > After fission, the "earlier" loop runs for ALL iterations before the "later" loop runs for ANY iteration. If the earlier stmt reads array `X` and the later stmt writes `X`, the earlier loop at iteration `i+1` reads the stale value that the later loop at iteration `i` should have updated.

      ```fortran
      ! Original — each sub-step feeds the next within the same t:
      do t = 1, tmax
        do j = 1, ny; ey(j, 1) = fict(t); end do                     ! S1: boundary
        do i = 2, nx; do j; ey(j,i) -= 0.5*(hz(j,i)-hz(j-1,i)); end; end  ! S2: reads hz
        do i; do j = 2, ny; ex(j,i) -= 0.5*(hz(j,i)-hz(j,i-1)); end; end  ! S3: reads hz
        do i; do j; hz(j,i) -= 0.7*(ex(j+1,i)+ey(j,i+1)-...); end; end    ! S4: writes hz
      end do

      ! After fission (WRONG):
      do t; [S1]; end do   ! loop 1
      do t; [S2]; end do   ! loop 2 — at t=2 reads hz(t=1), but loop 4 hasn't run yet
      do t; [S3]; end do   ! loop 3
      do t; [S4]; end do   ! loop 4 — hz written here, too late for loops 2-3
      ```

### 6.6.2 Transformation algorithm
  1. Detect a loop containing multiple independent statements or statement groups.
  2. Analyze dependencies between statements.
  4. Create two or more loops with the same iterator, bounds, and step.
  5. Move each statement group into the corresponding new loop.
  6. Replace the original loop with the generated sequence of loops.
### 6.6.3 Example
```fortran
 !Before:
 do i = 1, n
   a(i) = b(i)
   c(i) = d(i)
 end do

 !After:
 do i = 1, n
   a(i) = b(i)
 end do
 do i = 1, n
   c(i) = d(i)
 end do
```

## 6.7 Loop Unrolling

### 6.7.1 Preconditions
- innermost loop - no nested loops
### 6.7.2 Transformation algorithm
1. Find innermost loop for unrolling.
2. Read the unrolling factor.
3. Duplicate the loop body according to the unrolling factor.
4. Update index expressions for each duplicated copy.
5. Generate a remainder loop if the iteration count is not divisible by the factor.
6. Replace the original loop with the unrolled version.
### 6.7.3 Example
```fortran
 !Before (factor = 4):
 do i = 1, n
   a(i) = b(i)
 end do
 
 !After:
 do i = 1, n - 3, 4
   a(i) = b(i)
   a(i+1) = b(i+1)
   a(i+2) = b(i+2)
   a(i+3) = b(i+3)
 end do
 do i = n + 1 - MOD(n - (1) + 1, 4), n
   a(i) = b(i)
 end do
```

## 6.8 Loop Tiling
### 6.8.1 Preconditions
  - two nested loops
  - triangular inner bounds — inner bound references outer variable
    - ##### Triangular inner bound
      > The inner loop's lower bound depends on the outer loop variable. Tiling the outer pair `(j, i)` discards that dependency: the tile replaces the inner start with the tile-strip origin `ii`, but the original constraint `i >= j` is no longer enforced.
      ```fortran
      ! Before
      do j = 1, maxgrid
        do i = j, maxgrid        ! triangular: i starts at j, not 1
          do cnt = 1, length
            diff(cnt, i, j) = sumTang(i, j)
          end do
        end do
      end do

      ! After
      do jj = 1, maxgrid, 32
        do ii = j, maxgrid, 32     ! BUG 1: `j` is the inner tile variable — not yet defined here
          do j = jj, MIN(jj + 32 - 1, maxgrid)
            do i = ii, MIN(ii + 32 - 1, maxgrid)   ! BUG 2: should start at MAX(ii, j), not ii
              do cnt = 1, length
                diff(cnt, i, j) = sumtang(i, j)
              end do
            end do
          end do
        end do
      end do
      ```      
  - nested DO inside inner body uses outer variable in its bounds
    - ##### Nested loop in body references outer variable
    > A third loop nested in the body has a bound that depends on the outer tiled variable. Tiling reorders `(i, j)` tile pairs, so `j` values from a later tile (larger `jj`) can be processed before the matching `i` values are ready — producing reads of partially-accumulated array elements.
    
    ```fortran
    do i = 2, ni
      do j = 1, ni
        do k = 1, i - 1          ! k-bound references outer variable i
          b(j, i) = b(j, i) + alpha * a(k, i) * b(k, j)
        end do                    ! reads b(k, j): needs (i=j, j=k) to have run first
      end do
    end do

    !`b(k, j)` at row `k`, column `j` is *written* at iteration `(i=j, j=k)`, i.e. when
    !the outer loop reaches `i = j`. So for any read at `(i, j, k)` with `k < i`, the
    !value `b(k, j)` is valid only after outer iteration `i = j` has completed.

    ! After

    do ii = 2, ni, 32
      do jj = 1, ni, 32
        do i = ii, MIN(ii + 32 - 1, ni)
          do j = jj, MIN(jj + 32 - 1, ni)
            do k = 1, i - 1
              b(j, i) = b(j, i) + (alpha * a(k, i) * b(k, j))
            end do
          end do
        end do
      end do
    end do
    ```

### 6.8.2 Transformation algorithm
1. Detect loop with idealy nested loop
2. Check that loops' bounds have invariable trip count
3. Generate outer tile loops.
4. Generate inner point loops.
5. Adjust loop bounds to handle boundary tiles.
6. Move the original loop body into the innermost tiled loop.
### 6.8.3 Example
  ```fortran
 ! Before
  do i = lo_i, hi_i
    do j = lo_j, hi_j
      body
    end do
  end do
 ! After
  do ii = lo_i, hi_i, TILE
    do jj = lo_j, hi_j, TILE
      do i = ii, MIN(ii + TILE - 1, hi_i)
        do j = jj, MIN(jj + TILE - 1, hi_j)
          body
        end do
      end do
    end do
  end do
  ```
