## 5.1 Research Approach
The research follows an artifact-oriented approach in which a source-to-source transformation framework was designed and implemented for Fortran programs.

The developed framework extends the METAFOR infrastructure and provides automated support for loop fusion, loop fission, loop interchange, loop tiling, and loop unrolling.

The effectiveness of the proposed solution was evaluated through a set of benchmark programs representative of scientific and high-performance computing workloads.

## 5.3 Benchmark Selection
* Source of benchmarks - Polybench/Fortran
* Why they were chosen
* Selection criteria

Benchmarks were selected from scientific computing workloads written in Fortran.

The selected programs contain nested loops and array-intensive computations, making them suitable candidates for loop transformation techniques.

Benchmarks that did not contain transformable loop structures were excluded from the evaluation.

## 5.5 Correctness Validation
* Performed on small dataset
* Program compiles successfully.
* Original output == transformed output.
* Illegal transformations are rejected.
* To preserve program semantics, each transformation is guarded by a set of legality checks designed to detect dependency violations and unsafe loop structures.
